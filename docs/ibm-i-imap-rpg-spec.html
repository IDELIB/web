1. 概要

本ライブラリは、既存の ILE C 実装 (IBMi-imap/client/src/) を参照元として、 ILE RPG のみでフルスクラッチ再実装した RFC 3501 IMAP4rev1 クライアントライブラリです。 C コンパイラを必要とせず、IBM i 標準の BSD ソケット API と GSKit (Global Security Toolkit) を RPG から直接呼び出すことで、平文 (port 143) および IMAPS / STARTTLS (port 993) の両方に対応しています。

主な機能
  • TCP / IMAPS 接続 (GSKit TLS)
  • LOGIN / AUTHENTICATE PLAIN 認証
  • メールボックス SELECT / LIST / STATUS
  • UID SEARCH / FETCH / STORE / COPY
  • EXPUNGE / CAPABILITY
  • Base64 エンコード内蔵
設計ポリシー
  • RPG フルスクラッチ (C 依存なし)
  • 3 レイヤーモジュール分離
  • ヒープポインタ型コンテキスト
  • EBCDIC 対応 (大文字変換・バイト演算)
  • AUT(*EXCLUDE) セキュアデフォルト
品質保証
  • RPGUnit 単体テスト 49 件
  • 統合テスト 8 件 (実サーバー)
  • コードカバレッジ計測対応
  • testing.json でビルド管理
提供形式
  • サービスプログラム IMAPLIB
  • 3 モジュール構成
  • IFS ストリームファイル配布
  • QSYS メンバー変換対応

4. IMAPCONN モジュール — TCP/TLS 接続管理

レイヤー 1。BSD ソケット API と GSKit を使い、TCP 接続の確立・クローズ・データ送受信・ コマンドタグ生成・ソケット生存確認を担当します。 コンテキスト構造体 t_imapconn はヒープ上に確保され、ポインタで受け渡します。

手続き名戻り値概要主なパラメータ
imapconn_open int(10) TCP 接続確立 + (use_ssl=*on 時) GSKit TLS ハンドシェイク。接続後グリーティング 1 行を読み捨てる。 ctx・hostname・port・use_ssl・ssl_verify・timeout
imapconn_close int(10) GSKit セッション → GSKit 環境 → ソケットの順でクローズ。NULL ctx は無視。 ctx
imapconn_send int(10) IMAP コマンド文字列を送信。末尾に CRLF (x'0D0A') を自動付与。 ctx・cmd (varchar(8192))
imapconn_send_raw int(10) 生バイト列をそのまま送信 (リテラルブロック等)。CRLF 付与なし。 ctx・data (pointer)・len
imapconn_recv_line int(10) CRLF (\r\n) まで 1 行受信。内部バッファを使用しバイト単位で読み取る。 ctx・line (pointer)・maxLen
imapconn_next_tag varchar(64) コマンドタグを生成して返す。内部カウンタをインクリメント。A001, A002…A1000 形式。 ctx
imapconn_is_alive ind MSG_PEEK + MSG_DONTWAIT で recv() を呼びソケットの生存を確認。0 バイト受信 = 切断。 ctx
imapconn_get_error varchar(256) コンテキストに保存された最後のエラーメッセージ文字列を返す。 ctx
imapconn_get_error_code int(10) コンテキストに保存された最後のエラーコード (IMAP_ERR_*) を返す。 ctx
⚠
注意: imapconn_recv_line は 1 バイトずつ recv() を呼ぶ実装です。 大容量ボディ取得では呼び出し回数が増加します。IMAP プロトコルのコマンド/レスポンス交換 (行単位) での使用を前提としています。

5. IMAPRESP モジュール — レスポンスパーサー

レイヤー 2。IMAP サーバーから受信した生テキスト行を解析し、 タグ付きレスポンス (例: A001 OK)・アンタグドレスポンス (例: * 10 EXISTS)・ 継続レスポンス (+) を識別して t_imap_parsed_resp 構造体に格納します。

手続き名戻り値概要
imapresp_parse_line int(10) (0=成功 / -1=エラー) 生テキスト 1 行を解析して t_imap_parsed_resp に格納。キーワードは EBCDIC 大文字変換 (_to_upper) を適用。
imapresp_wait_tagged int(10) IMAP_RESP_* 指定タグの完了レスポンス (OK/NO/BAD) まで行を読み続け、アンタグドラインを t_imap_lines に蓄積する。IMAPCLI の全コマンドが内部で使用。
imapresp_lines_init ― t_imap_lines の動的配列を初期化 (%alloc)。容量 RESP_LINES_INIT_CAP (16) で開始。
imapresp_lines_free ― t_imap_lines の動的配列を解放 (%dealloc)。count / capacity を 0 にリセット。
imapresp_lines_get int(10) (0=成功 / -1=範囲外) N 番目 (0-based) のアンタグドラインを lineOut バッファにコピーして返す。

t_imap_parsed_resp フィールド

フィールド型値
type int(10)RESP_TYPE_STATUS(0) / RESP_TYPE_DATA(1) / RESP_TYPE_CONT(2)
tag char(64)タグ文字列 (アンタグドは '*')
resp_code int(10)IMAP_RESP_OK(0) / NO(1) / BAD(2) / BYE(3) / CONTINUE(4)
keyword char(32)大文字キーワード (EXISTS / FETCH / LIST 等)
number int(10)数値 (-1 = なし)。例: * 10 EXISTS の 10
text char(8192)キーワード以降の残テキスト全体
is_untaggedind アンタグドフラグ (*on = '*' 始まり)

6. IMAPCLI モジュール — 高レベルクライアント API

レイヤー 3。呼び出し元 RPG プログラムが直接使用するパブリック API 層です。 IMAPCONN と IMAPRESP の上に IMAP4rev1 の各コマンドを実装し、 接続状態管理・コマンド組み立て・レスポンス解析・結果データ構造体への格納を一括して行います。

6.1 ライフサイクル管理

手続き名戻り値概要
imap_client_init pointer デフォルト設定 (timeout=30, ssl_verify=*on, max_retries=3) でクライアントコンテキストを確保・初期化して返す。内部で imap_client_init_cfg を呼ぶ。
imap_client_init_cfg pointer t_imap_client_config を受け取りカスタム設定で初期化。t_imap_client と t_imapconn を %alloc で確保する。失敗時は *null を返す。
imap_client_cleanup ― 接続中の場合は LOGOUT を試み、全ヒープリソース (conn_ctx・cliPtr) を %dealloc で解放する。NULL 引数は安全に無視。

6.2 接続 API

手続き名戻り値必要状態概要
imap_connect int(10) DISCONNECTED TCP/TLS 接続を確立し CAPABILITY コマンドを発行してサーバー機能を取得。成功後は CONNECTED 状態に遷移。
imap_disconnect int(10) CONNECTED 以上 LOGOUT コマンドなしでソケットを即時クローズ。緊急切断時に使用。
imap_is_connected ind ― state が CONNECTED (2) 以上なら *on を返す。NULL ctx は *off。

6.3 認証 API

手続き名戻り値必要状態概要
imap_login int(10) CONNECTED TAG LOGIN username password コマンドを送信。成功後は AUTHENTICATED 状態に遷移。LOGINDISABLED キャパビリティが設定されている場合は非推奨。
imap_authenticate_plain int(10) CONNECTED TAG AUTHENTICATE PLAIN を送信し、サーバーの継続要求 (+) に対して \0user\0pass 形式の Base64 エンコード文字列で応答する。内蔵 _base64_encode を使用。
imap_logout int(10) CONNECTED 以上 TAG LOGOUT コマンドを送信して正常切断。成功後は DISCONNECTED 状態に遷移。

6.4 メールボックス API

手続き名戻り値必要状態概要
imap_select int(10) AUTHENTICATED TAG SELECT mailbox。成功後は SELECTED 状態。t_imap_mailbox に exists / recent / uidvalidity / uidnext を格納。
imap_examine int(10) AUTHENTICATED TAG EXAMINE mailbox。読み取り専用 SELECT。SELECT と同じ情報を格納するが書き込みコマンドは拒否される。
imap_list int(10) AUTHENTICATED TAG LIST ref pattern。t_imap_mbox_list にメールボックス情報を格納。呼び出し元が imap_mbox_list_free で解放する責任を持つ。
imap_status int(10) AUTHENTICATED TAG STATUS mailbox (items)。items 例: 'MESSAGES UNSEEN UIDNEXT'。t_imap_mailbox に統計値を格納。
imap_create int(10) AUTHENTICATED TAG CREATE mailbox。新規メールボックスを作成。
imap_delete_mbox int(10) AUTHENTICATED TAG DELETE mailbox。メールボックスを削除。
imap_rename_mbox int(10) AUTHENTICATED TAG RENAME oldname newname。メールボックスをリネーム。

6.5 メッセージ API

手続き名戻り値必要状態概要
imap_search int(10) SELECTED TAG UID SEARCH criteria。criteria 例: 'UNSEEN'、'FROM "user@example.com"'。t_imap_uid_list に一致 UID を格納。呼び出し元が imap_uid_list_free で解放する。
imap_fetch int(10) SELECTED TAG UID FETCH uid (items)。items 例: 'FLAGS RFC822.SIZE BODY[HEADER.FIELDS (FROM TO SUBJECT)]'。t_imap_message に結果を格納。
imap_store_flags int(10) SELECTED TAG UID STORE uid mode FLAGS (flagList)。mode: '+FLAGS'(追加) / '-FLAGS'(削除) / 'FLAGS'(上書き)。flags は IMAP_FLAG_* ビット組み合わせを渡す。
imap_copy_msg int(10) SELECTED TAG UID COPY uid destMailbox。別メールボックスへメッセージをコピー。
imap_expunge int(10) SELECTED TAG EXPUNGE。\Deleted フラグが付いたメッセージを完全削除。

6.6 ユーティリティ API

手続き名戻り値概要
imap_get_capabilities uns(10) TAG CAPABILITY を発行し IMAP_CAP_* ビットマスクを返す。%bitand で個別機能を確認可能。
imap_get_error varchar(256) クライアントコンテキストに保存された最後のエラーメッセージを返す。
imap_get_error_code int(10) 最後のエラーコード (IMAP_ERR_*) を返す。
imap_uid_list_free ― t_imap_uid_list の uids_ptr を %dealloc して count / capacity をリセット。
imap_mbox_list_free ― t_imap_mbox_list の items_ptr を %dealloc して count / capacity をリセット。

7. メモリ管理

コンテキスト確保解放責任
t_imap_client (IMAPCLI_CTX_SIZE ≈ 900 B) imap_client_init / imap_client_init_cfg imap_client_cleanup ライブラリ
t_imapconn (IMAPCONN_CTX_SIZE ≈ 8784 B) imap_client_init 内で自動確保 imap_client_cleanup ライブラリ

8. TLS ハンドシェイクフロー (GSKit)

IBM i Digital Certificate Manager (DCM) を使用した証明書管理が前提です。

1. gsk_environment_open()               → GSKit 環境ハンドル取得
2. gsk_attribute_set_enum               → GSK_SESSION_TYPE = GSK_CLIENT_SESSION
3. gsk_attribute_set_enum               → GSK_SERVER_AUTH_TYPE
                                           ssl_verify=*on  → GSK_SERVER_AUTH_FULL  (完全証明書検証)
                                           ssl_verify=*off → GSK_SERVER_AUTH_PASSTHRU (検証スキップ)
4. gsk_environment_init()               → TLS 設定確定

5. socket(AF_INET, SOCK_STREAM, 0)      → TCP ソケット作成
6. setsockopt(SO_RCVTIMEO / SO_SNDTIMEO)→ タイムアウト設定 (SOL_SOCKET = -1 on IBM i)
7. connect(fd, sockaddr_in, ...)        → TCP 接続確立

8. gsk_secure_soc_open(env_handle)      → GSKit セッションハンドル取得
9. gsk_attribute_set_numeric_value(GSK_FD, fd) → ソケット fd を関連付け
10. gsk_secure_soc_init(soc_handle)     → TLS ハンドシェイク実行

   ── 以降は gsk_secure_soc_write / gsk_secure_soc_read で通信 ──

11. gsk_secure_soc_close(soc_handle)    → TLS セッション終了
12. gsk_environment_close(env_handle)   → GSKit 環境終了
13. close(fd)                           → TCP ソケットクローズ

9. コンパイル手順

9.1 IFS ストリームファイルからコンパイル (推奨)

依存順序どおりに 3 モジュールをコンパイルしてから SRVPGM を作成します。

/* ── Step 1: IMAPCONN モジュール ─────────────────────────── */
CRTRPGMOD MODULE(MYLIB/IMAPCONN)
          SRCSTMF('/IBMi-imap/rpg/src/IMAPCONN.rpgle')
          OPTION(*EVENTF *SRCSTMT)
          DBGVIEW(*SOURCE)
          INCDIR('/IBMi-imap/rpg/includes')

/* ── Step 2: IMAPRESP モジュール ─────────────────────────── */
CRTRPGMOD MODULE(MYLIB/IMAPRESP)
          SRCSTMF('/IBMi-imap/rpg/src/IMAPRESP.rpgle')
          OPTION(*EVENTF *SRCSTMT)
          DBGVIEW(*SOURCE)
          INCDIR('/IBMi-imap/rpg/includes')

/* ── Step 3: IMAPCLI モジュール ──────────────────────────── */
CRTRPGMOD MODULE(MYLIB/IMAPCLI)
          SRCSTMF('/IBMi-imap/rpg/src/IMAPCLI.rpgle')
          OPTION(*EVENTF *SRCSTMT)
          DBGVIEW(*SOURCE)
          INCDIR('/IBMi-imap/rpg/includes')

/* ── Step 4: サービスプログラム作成 ─────────────────────── */
CRTSRVPGM SRVPGM(MYLIB/IMAPLIB)
          MODULE(MYLIB/IMAPCONN MYLIB/IMAPRESP MYLIB/IMAPCLI)
          EXPORT(*ALL)
          BNDDIR(QC2LE QSYS/QSOSSLBND)
          ACTGRP(*CALLER)
          AUT(*EXCLUDE)
✓
QSOSSLBND バインディングディレクトリには GSKit ライブラリが含まれています。 AUT(*EXCLUDE) により、明示的な権限付与なしに他ユーザーからのアクセスを拒否するセキュアデフォルト設定です。

11. クイックスタートサンプル

INBOX の未読メッセージを検索して先頭メールのヘッダーを取得する最小サンプルです。

ILE RPG**free
/include 'IBMi-imap/rpg/includes/IMAPH.rpgleinc'

// ── プロトタイプ宣言 ──────────────────────────────────────
dcl-pr imap_client_init    pointer         extproc('imap_client_init');    end-pr;
dcl-pr imap_client_cleanup                extproc('imap_client_cleanup');
  cliPtr  pointer value;  end-pr;
dcl-pr imap_connect        int(10)         extproc('imap_connect');
  cliPtr  pointer value;
  host    varchar(256) const;
  port    int(10) value;
  ssl     ind value;      end-pr;
dcl-pr imap_login          int(10)         extproc('imap_login');
  cliPtr  pointer value;
  user    varchar(256) const;
  pass    varchar(256) const;  end-pr;
dcl-pr imap_select         int(10)         extproc('imap_select');
  cliPtr  pointer value;
  mbox    varchar(256) const;
  mboxInfo likeds(t_imap_mailbox);  end-pr;
dcl-pr imap_search         int(10)         extproc('imap_search');
  cliPtr  pointer value;
  crit    varchar(256) const;
  uids    likeds(t_imap_uid_list);  end-pr;
dcl-pr imap_fetch          int(10)         extproc('imap_fetch');
  cliPtr  pointer value;
  uid     int(10) value;
  items   varchar(128) const;
  msg     likeds(t_imap_message);  end-pr;
dcl-pr imap_logout         int(10)         extproc('imap_logout');
  cliPtr  pointer value;  end-pr;
dcl-pr imap_get_error      varchar(256)    extproc('imap_get_error');
  cliPtr  pointer value;  end-pr;
dcl-pr imap_uid_list_free               extproc('imap_uid_list_free');
  uids    likeds(t_imap_uid_list);  end-pr;

// ── 変数宣言 ──────────────────────────────────────────────
dcl-s  cliPtr  pointer;
dcl-ds inbox   likeds(t_imap_mailbox) inz;
dcl-ds uids    likeds(t_imap_uid_list) inz;
dcl-ds msg     likeds(t_imap_message)  inz;
dcl-s  rc      int(10);
dcl-s  uidPtr  pointer;
dcl-s  firstUid int(10) based(uidPtr);

// ── 初期化 & 接続 ────────────────────────────────────────
cliPtr = imap_client_init();

rc = imap_connect(cliPtr : 'imap.example.com' : 993 : *on);
if rc <> IMAP_ERR_NONE;
  dsply imap_get_error(cliPtr);
  imap_client_cleanup(cliPtr);
  *inlr = *on;
  return;
endif;

// ── LOGIN 認証 ───────────────────────────────────────────
rc = imap_login(cliPtr : 'user@example.com' : 'password');
if rc <> IMAP_ERR_NONE;
  dsply imap_get_error(cliPtr);
  imap_client_cleanup(cliPtr);
  *inlr = *on;
  return;
endif;

// ── INBOX SELECT & 未読検索 ──────────────────────────────
rc = imap_select(cliPtr : 'INBOX' : inbox);
// inbox.exists = 総メッセージ数, inbox.unseen = 未読数

rc = imap_search(cliPtr : 'UNSEEN' : uids);
if rc = IMAP_ERR_NONE and uids.count > 0;
  uidPtr = uids.uids_ptr;
  rc = imap_fetch(cliPtr : firstUid
               : 'FLAGS RFC822.SIZE BODY[HEADER.FIELDS (FROM TO SUBJECT)]'
               : msg);
  // msg.subject / msg.from_addr / msg.size を参照
endif;
imap_uid_list_free(uids);   // ← 呼び出し元責任で解放

// ── 後処理 ──────────────────────────────────────────────
imap_logout(cliPtr);
imap_client_cleanup(cliPtr);
*inlr = *on;

12. IBM i 固有の実装注意点

項目問題本実装の解決策
SOL_SOCKET の値 Linux/AIX では 1 だが IBM i では -1 IMAPH.rpgleinc で dcl-c SOL_SOCKET -1 として明示定義
recv フラグの組み合わせ MSG_PEEK + MSG_DONTWAIT の加算は誤り %bitor(MSG_PEEK : MSG_DONTWAIT) でビット OR を使用
バイト値取得 (EBCDIC) %dec(char:1:0) は EBCDIC コードポイントで誤動作 uns(3) based(bytePtr) でポインタ演算して 1 バイト読み取り
%right BIF IBM i RPG に %right BIF は存在しない if tagNum <= 9 / <= 99 / <= 999 の条件分岐でゼロ埋め実装
varchar → C API 引数 %addr(varchar) は 2 バイトの長さフィールドを指す hostnameZ = %trimr(hostname) + x'00' で null 終端 char に変換
EBCDIC 大文字変換 %UPPER や C の toupper は EBCDIC では信頼性が低い _to_upper 手続きで IMAP_LOWER_ALPHA → IMAP_UPPER_ALPHA の手動マップ変換
Base64 バイト演算 %dec による文字コード算術は EBCDIC で不正 byteRead uns(3) based(bytePtr) でポインタオフセット移動して読み取り
%scan の文字セット検索 %scan(ch:'0123456789')-1 は EBCDIC で誤変換リスク ch >= '0' and ch <= '9' の比較式に変更 (EBCDIC 連続性を保証)
⚠
IBM i の EBCDIC 環境では ASCII 前提のコードがサイレントに誤動作します。 文字コード算術・大文字変換・バイト値取得には上記の対策を必ず適用してください。
