1. 概要

TN5250Session は IBM Host Access Class Library (HACL / ECL) を使って IBM i (AS/400) の TN5250 端末セッションを制御する Java プログラムです。 index.ts（Node.js MCP サーバー）に子プロセスとして起動され、 stdin から受け取るテキストコマンドに従って画面操作を行い、結果を stdout へ出力します。

index.tsNode.js / MCP Server
↕stdin (コマンド) / stdout (テキスト応答)
TN5250Session.javajava -jar TN5250Session.jar
↕IBM ECL API (HACL)
ECLSession / ECLPScom.ibm.eNetwork.ECL.*
↕TCP TN5250 / Telnet (port 23)
IBM i ホストAS/400 サーバー

OIA.WaitForInput() を最大3回リトライ（合計最大18秒）。 ネットワーク遅延やホスト側の処理遅延に対する堅牢性を確保しています。
private signOn(user, password) 162〜179行

サインオン画面にユーザー名とパスワードを送信します。

169ps.SendKeys(user + "[tab]" + password + "[enter]");

[tab] でパスワードフィールドへ移動し、[enter] でサインオンを実行します。 この1回の SendKeys 呼び出しで、ユーザー名入力・タブ移動・パスワード入力・Enter 送信をまとめて行います。

ユーザー名とパスワードは引数経由で渡されます。Node.js 側（index.ts）がコマンドライン引数として渡すため、 プロセスリストに表示される可能性があります。本番環境での運用時は注意が必要です。

7. エラーハンドリング設計

セキュリティを意識した例外処理の一貫したパターン

すべての例外キャッチ箇所で スタックトレースを stdout/stderr に直接出力しない 設計になっています。

  • ユーザー向けには「Please contact the system administrator.」のような汎用メッセージを出力
  • 詳細は java.util.logging.Logger に記録（ログファイルのみ）
  • これにより、MCP 経由で AI がエラー詳細（内部パスやスタック）を取得できない設計
場所ログレベル想定状況
main() catchSEVERE接続・サインオン・全般的な例外
sendKeys() ECLErr catchWARNING無効なキーシーケンス
sendKeys() Exception catchSEVEREその他の予期しない例外
interactiveMode() catchSEVEREコマンド実行中の例外
displayScreen() catchWARNING画面取得エラー
displayFieldInfo() catchSEVEREフィールドリスト取得エラー

8. Node.js (index.ts) との通信プロトコル

stdin/stdout による独自テキストプロトコル
方向内容例
Node.js → Java (stdin)1行テキストコマンドsend WRKSYSSTS[enter]\n
Java → Node.js (stdout)複数行テキスト応答画面テキスト・フィールド情報など
Java → Node.js (stderr)エラー・警告メッセージWarning: Input not available

Node.js 側は time-based polling（待機時間後にバッファを読む）で応答を収集します。 コマンド終了を示すデリミタはないため、wait_ms の設定が応答の完全性に影響します。
