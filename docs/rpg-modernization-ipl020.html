2. プログラム概要

IPL020 は品目照会（サブファイル表示）プログラムです。 WORKSTNデバイスファイル IPL020D を使用してサブファイル画面を表示し、 外部記述キーファイル HINMSL01（品目マスター）からデータを読み込んでサブファイルに書き出します。

項目内容
H-spec（制御仕様）DATEDIT(*YMD)
F-spec（ファイル）IPL020D（WORKSTN/CF/E, SFL付き）、HINMSL01（DISK/IF/E/K）
D-spec（変数定義）#NAKN（LIKE(HNNAKN)）、#CTL（CHAR 4）、#RRN01（PACKED 4:0）
C-spec（ロジック）CASE ディスパッチ、サブルーチン 5本（@HED, @DTL, @CHKH, @SET, @END）、初期化 *INZSR
インジケーター*IN03（F3キー）、*IN32（SFLEND）、*IN33（SFLクリア）、*IN90（SETLL結果）、*IN91（EOF）、*IN92（WRITEエラー）、*INLR

3. 仕様タイプ別 変換内容

H-spec → CTL-OPT

変換前（固定フォーマット）変換後（フリーフォーム）
H DATEDIT(*YMD)CTL-OPT DATEDIT(*YMD);

プログラム先頭に **FREE 指示子を追加。

F-spec → DCL-F

変換前（固定）変換後（フリーフォーム）備考
FIPL020D CF E WORKSTN SFILE(SFL01:#RRN01) DCL-F IPL020D WORKSTN SFILE(SFL01:#RRN01); WORKSTNデバイスキーワードを最初に配置
FHINMSL01 IF E K DISK DCL-F HINMSL01 KEYED USAGE(*INPUT); DISK(*EXT) はデフォルトのため省略、KEYED + USAGE(*INPUT) に変換
注意: 初回コンパイルでは DCL-F ipl020d（小文字）を指定したところ、CCSID 5026（日本語）環境で小文字英字が特殊文字として解釈され文字化けが発生。ファイル名・変数名をすべて大文字に修正することで解決。

D-spec → DCL-S

変換前（固定）変換後（フリーフォーム）
D #NAKN S LIKE(HNNAKN)DCL-S #NAKN LIKE(HNNAKN);
D #CTL S 4DCL-S #CTL CHAR(4);
D #RRN01 S 4 0DCL-S #RRN01 PACKED(4:0);

C-spec → フリーフォーム

CAS/ENDCS → SELECT/WHEN/ENDSL

// 変換前 C #CTL CASEQ 'HED ' @HED C #CTL CASEQ 'DTL ' @DTL C #CTL CASEQ 'END ' @END C ENDCS // 変換後 SELECT; WHEN #CTL = 'HED '; EXSR @HED; WHEN #CTL = 'DTL '; EXSR @DTL; WHEN #CTL = 'END '; EXSR @END; ENDSL;

EVAL → 直接代入

// 変換前 C EVAL #CTL = 'END ' // 変換後 #CTL = 'END ';

インジケーター結果の変換（BIF化）

変換前（固定・結果インジケーター）変換後（フリーフォーム BIF）
SETLL #NAKN HINMSL01  90 SETLL #NAKN HINMSL01;
IF NOT %FOUND(HINMSL01); *IN90 = *ON; ELSE; *IN90 = *OFF; ENDIF;
READ HINMSL01  91 READ HINMSL01;
*IN91 = %EOF(HINMSL01);
WRITE SFL01  92 WRITE SFL01;（エラーインジケーター92は保持）

インジケーター一覧

インジケーター用途変換方法
*IN03F3キー（終了）IF *IN03 = *ON; として保持
*IN32SFLENDフラグ*IN32 = *ON/OFF; として保持
*IN33サブファイルクリア*IN33 = *ON/OFF; として保持
*IN90SETLL失敗（Not Found）IF NOT %FOUND(HINMSL01); で設定
*IN91READ EOF%EOF(HINMSL01) の結果を代入
*IN92SFLWRITEエラーインジケーターとして保持
*INLR最終レコード（終了）*INLR = *ON; として保持

4. コンパイル実行ログ

Step 4 — 事前コンパイル（Fixed Format）

✅ 成功試行 1/1
コマンド: CRTBNDRPG PGM(MYLIB/IPL020_T) SRCFILE(MYLIB/QRPGLESRC)
修正: なし

Step 9 — フリーフォーム変換後コンパイル

❌ 失敗試行 1/2
コマンド: CRTBNDRPG PGM(MYLIB/IPL020_FRE) SRCFILE(MYLIB/QRPGLESRC)
エラー原因:
  • 文字化け問題: ファイル名を小文字で指定（ipl020d）したところ、CCSID 5026環境で特殊文字として変換されコンパイラーが認識できなかった
  • RNF2368: DISK(*EXT) キーワードの位置が不正（最初のキーワードに配置する必要あり）
  • RNF0391/0394: %EOF/%FOUND のパラメーターが小文字のため未解決
  • RNF7030: ファイル名未解決に伴うインジケーター未定義エラー
✅ 成功試行 2/2（自己修復後）
修正内容:
  • ファイル名・変数名をすべて大文字に変更（IPL020D、HINMSL01）
  • DCL-F HINMSL01 KEYED USAGE(*INPUT);（DISK(*EXT) を削除 — デフォルトのため不要）
  • %FOUND(HINMSL01)、%EOF(HINMSL01) を大文字化
  • SETLL の後処理ロジックを IF NOT %FOUND / ELSE 構文に修正
コマンド: CRTBNDRPG PGM(MYLIB/IPL020_FRE) SRCFILE(MYLIB/QRPGLESRC)
結果: プログラムオブジェクト MYLIB/IPL020_FRE (*PGM) 作成成功
